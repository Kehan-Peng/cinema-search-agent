# 智能语义搜索 Agent 功能

## 任务目标
在保持现有推荐功能的基础上，增加一个新的智能搜索 agent 功能，实现：
1. 对海报的模糊智能查询（如"一个男人站在大雨里" -> 肖申克的救赎）
2. 对情节的模糊智能查询（如"逃出自己建设的监狱" -> 肖申克的救赎）

## 现状分析

### 现有能力
- 项目已有语义嵌入能力（`myutils/recommender/semantic_embeddings.py`）
- 支持 Word2Vec 和 GloVe 两种本地轻量级语义模型
- 现有搜索功能仅支持关键词匹配（标题、类型、导演、国家）

### 技术方案
由于项目定位是"单机运行、无服务器部署"，需要采用轻量级方案：

**方案选择：**
1. **基础方案**：扩展现有语义嵌入能力，对电影的 summary（剧情简介）进行语义编码
2. **增强方案**：可选接入 LLM API（项目已有 `MOVIE_LLM_BASE_URL` 配置）进行更精准的语义理解

**实现路径：**
- 复用现有的 `semantic_embeddings.py` 中的语义编码能力
- 新增查询文本的语义编码功能
- 实现查询向量与电影向量的相似度匹配
- 在前端增加智能搜索入口

## 实施计划

### 阶段一：后端语义搜索能力
- [ ] 在 `myutils/recommender/semantic_embeddings.py` 中新增查询文本编码函数
- [ ] 新增语义搜索函数，支持对剧情和海报描述的模糊查询
- [ ] 在 `repositories/movie_repository.py` 中新增语义搜索接口

### 阶段二：API 路由与前端集成
- [ ] 在 `app.py` 中新增语义搜索 API 路由

- [ ] 添加搜索结果的相似度得分展示

### 阶段三：数据增强（可选）
- [ ] 为现有电影数据补充更详细的剧情描述
- [ ] 如果需要海报描述，可以考虑接入 LLM 生成海报场景描述

## 技术细节

### 语义编码策略
1. 对用户查询文本进行分词和标准化
2. 使用现有的 Word2Vec/GloVe 模型生成查询向量
3. 计算查询向量与所有电影向量的余弦相似度
4. 返回相似度最高的 top-k 结果

### 数据库扩展
考虑是否需要在 `movies` 表中新增字段：
- `poster_description`：海报场景描述（用于海报搜索）
- 或直接使用现有的 `summary` 字段

## 风险与约束
- 本地轻量级语义模型的准确度有限，对于复杂的语义理解可能不够精准
- 如果电影数据的 summary 字段内容不够丰富，搜索效果会受限
- 需要权衡搜索准确度和单机运行的性能开销
